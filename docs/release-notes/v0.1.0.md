## What this is

`gateway-benchmarks` measures seven production API gateways
(**NGINX, Envoy, Wallarm, Traefik, Kong, APISIX, Tyk**) under the
**same policy × load × scenario** matrix, with parity attestation
before every cell and a reproducibility gate between any two runs.

Everything — configs, fixtures, seeds, image digests, orchestrator,
report renderer — is frozen at this tag. Run it yourself, locally or
on a 3-host AWS cluster placement group, and you get a
byte-for-byte equivalent ranking within the documented tolerance.

## How to reproduce

Locally (smoke; does **not** produce canonical numbers):

```bash
git clone https://github.com/wallarm/gateway-benchmarks
cd gateway-benchmarks
git checkout v0.1.0
make perf-local-up
make perf-local-run
make perf-local-report
```

AWS canonical (produces bit-equivalent numbers within tolerance):

```bash
git checkout v0.1.0
# follow docs/CANONICAL-RUN-HANDOFF.md — the entire ceremony is
# a Makefile one-liner with a 10-minute pre-flight checklist.
```

## Changes since initial scaffolding

See [`CHANGELOG.md § 0.1.0`](../../CHANGELOG.md) for the detailed list;
the headline items are:

- Full benchmark matrix: **7 gateways × 12 policy profiles × 4 load
  profiles × 11 HTTP scenarios**, plus the supplemental
  `p03-jwks-rs256-basic` capability track.
- Parity attestation on every cell.
- Native Go orchestrator (`bench`) with manifest pinning, atomic
  checkpoint, crash watchdog, retry-on-crash, and a native
  Docker-Engine stats collector (drops the shell sidecar dependency).
- Self-contained HTML report (`bench report`) rendered straight from
  `cells.jsonl + manifest.json`.
- Reproducibility gate (`bench compare-runs`) with exit codes
  `0 REPRODUCIBLE · 1 SOFT DIFF · 2 NOT REPRODUCIBLE`.
- Local + AWS infrastructure (OpenTofu cluster placement group).

## Neutrality reminder

Wallarm, Inc. maintains this repository and ships one of the gateways
under test. We neutralise the conflict of interest by freezing every
config at release, enforcing parity attestation before every cell,
and accepting reasonable external tuning of competing gateways as a
PR — see [`CONTRIBUTING.md`](../../CONTRIBUTING.md).

The benchmark is **not a tool for making a particular gateway look
good**. It is a tool for letting an independent reviewer reproduce
the ranking bit-for-bit and form their own opinion.
